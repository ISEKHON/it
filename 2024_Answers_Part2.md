**Step-by-Step Process:**
1. **User types URL** (www.google.com) in browser
2. **DNS Lookup:** DNS server converts domain to IP address (142.250.XXX.XXX)
3. **Request sent:** Browser sends HTTP request to server
4. **Routing:** Routers find best path across networks
5. **Data Packets:** Data broken into small packets
6. **Transmission:** Packets travel via different routes
7. **Reassembly:** Server/client reassembles packets
8. **Response:** Server sends webpage data back
9. **Display:** Browser renders HTML/CSS/JavaScript

**3. Key Protocols:**
- **TCP/IP:** Transmission Control Protocol/Internet Protocol (foundation)
- **HTTP/HTTPS:** Web page transfer (secure version with encryption)
- **FTP:** File Transfer Protocol
- **SMTP/POP3:** Email sending/receiving
- **DNS:** Domain Name System (converts names to IPs)

### 6. What is the difference between Primary and Secondary Memory? Explain the various types of primary memories. (4+11=15 marks)

**Answer:**

#### Difference between Primary and Secondary Memory (4 marks)

| Aspect | Primary Memory | Secondary Memory |
|--------|----------------|------------------|
| **Volatility** | Volatile (RAM) / Non-volatile (ROM) | Non-volatile (permanent) |
| **Speed** | Very fast (nanoseconds) | Slower (milliseconds) |
| **Capacity** | Small (4-64 GB typical) | Large (500 GB - 10 TB) |
| **Cost** | Expensive per GB | Cheap per GB |
| **CPU Access** | Directly accessed by CPU | Accessed via I/O channels |
| **Purpose** | Stores current programs/data | Long-term storage |
| **Examples** | RAM, ROM, Cache | HDD, SSD, USB drives |
| **Data retention** | Temporary (RAM) or permanent (ROM) | Permanent |

---

#### Types of Primary Memory (11 marks)

## 1. RAM (Random Access Memory) - 5 marks

**Definition:**
Volatile memory that temporarily stores data and programs currently being processed.

**Characteristics:**
- **Volatile:** Data lost when power off
- **Read/Write:** Can both read and write
- **Random Access:** Any location accessed directly
- **Fast:** Nanosecond access time
- **Temporary:** Holds only active work

**Types of RAM:**

### a) DRAM (Dynamic RAM) - 2.5 marks
**Technology:** Uses capacitors to store data (needs constant refresh)

**Features:**
- Must be refreshed thousands of times per second
- Cheaper than SRAM
- Slower than SRAM
- Higher density (more storage in same space)
- Used for main system memory

**Generations:**
- **SDRAM:** Synchronous DRAM (synchronized with CPU)
- **DDR:** Double Data Rate (transfers data twice per clock cycle)
- **DDR2, DDR3, DDR4, DDR5:** Progressively faster and efficient

**Capacity:** 4 GB, 8 GB, 16 GB, 32 GB, 64 GB modules

**Usage:** Computer's main RAM (system memory)

---

### b) SRAM (Static RAM) - 2.5 marks
**Technology:** Uses flip-flops (transistors) to store data

**Features:**
- No refresh needed (static)
- Very fast (faster than DRAM)
- More expensive
- Lower density (less storage per chip)
- Consumes less power
- Used for cache memory

**Types of Cache:**
- **L1 Cache:** 32-128 KB per core (fastest, inside CPU)
- **L2 Cache:** 256 KB - 1 MB per core
- **L3 Cache:** 4-32 MB (shared among cores)

**Usage:** CPU cache, high-speed buffers

**Why More RAM Helps:**
- Run more programs simultaneously
- Faster multitasking
- Better gaming/video editing performance
- Fewer slowdowns and freezing

---

## 2. ROM (Read-Only Memory) - 3 marks

**Definition:**
Non-volatile memory with pre-written data that persists even when power is off.

**Characteristics:**
- **Non-volatile:** Retains data without power
- **Read-only:** Generally cannot be modified (or very difficult)
- **Permanent:** Contains permanent instructions
- **Small capacity:** KB to few MB
- **Purpose:** Stores firmware and boot instructions

**Uses:**
- **BIOS/UEFI:** Basic Input/Output System (starts computer)
- **Firmware:** Permanent software in devices
- **Bootstrap loader:** Initial program to load OS
- **Embedded systems:** Calculators, washing machines, microwaves

---

**Types of ROM:**

### a) Mask ROM (MROM)
- **Programmed:** During manufacturing (factory-set)
- **Modification:** Cannot be changed
- **Cost:** Cheap for mass production
- **Use:** Old calculators, game cartridges
- **Status:** Obsolete

### b) PROM (Programmable ROM)
- **Programming:** User can write once using PROM programmer
- **Modification:** Cannot be erased (one-time only)
- **Use:** Custom firmware, product serial numbers
- **Analogy:** Like burning a CD-R (write once)

### c) EPROM (Erasable PROM)
- **Erase method:** UV (ultraviolet) light exposure (20-40 minutes)
- **Erase window:** Quartz window on chip for UV exposure
- **Reprogramming:** Multiple times (after erasing)
- **Drawback:** Must remove from circuit to erase
- **Use:** Development, prototyping
- **Status:** Largely obsolete (replaced by EEPROM)

### d) EEPROM (Electrically Erasable PROM)
- **Erase method:** Electrical signals (no UV light)
- **Convenience:** Can erase without removing from circuit
- **Granularity:** Byte-level erase (individual bytes)
- **Speed:** Slow write, fast read
- **Write cycles:** ~100,000 to 1 million
- **Use:** BIOS chips, storing settings, calibration data
- **Example:** System BIOS, network card MAC addresses

### e) Flash Memory
- **Type:** Special EEPROM with block-level erase
- **Speed:** Faster than EEPROM
- **Capacity:** GB scale (high density)
- **Erase:** Block-by-block (faster than byte-by-byte)
- **Types:**
  - **NOR Flash:** Random access, slower write (firmware)
  - **NAND Flash:** Sequential access, faster write (storage)
- **Use:** 
  - USB flash drives
  - Memory cards (SD, microSD)
  - SSDs
  - Smartphone storage
  - Digital cameras

**Advantages of Flash:**
- No moving parts (durable)
- Shock-resistant
- Low power consumption
- Silent operation
- Portable

---

## 3. Cache Memory - 3 marks

**Definition:**
Very small, ultra-fast memory between CPU and RAM that stores frequently accessed data.

**Purpose:**
- Reduce CPU wait time for data
- Bridge speed gap between fast CPU and slower RAM
- Improve overall system performance

**Hierarchy (Fastest â†’ Slowest):**

**L1 Cache:**
- **Size:** 32-128 KB per core
- **Speed:** 1-2 CPU clock cycles
- **Location:** Inside CPU core
- **Split:** Separate for instructions (I-cache) and data (D-cache)

**L2 Cache:**
- **Size:** 256 KB - 1 MB per core
- **Speed:** 4-12 clock cycles
- **Location:** Inside or very close to CPU

**L3 Cache:**
- **Size:** 4-32 MB
- **Speed:** 20-40 clock cycles
- **Sharing:** Shared among all CPU cores

**How Cache Works:**
1. CPU needs data
2. Checks L1 cache first (fastest)
3. If found (cache hit) â†’ Use data immediately
4. If not found (cache miss) â†’ Check L2 cache
5. If not in L2 â†’ Check L3 cache
6. If not in L3 â†’ Fetch from RAM (slow)
7. Copy data to cache for future use

**Cache Hit vs Miss:**
- **Cache Hit:** Data found in cache (fast, ~1-2 cycles)
- **Cache Miss:** Data not in cache, must fetch from RAM (slow, ~100+ cycles)

**Benefits:**
- Dramatically speeds up repetitive tasks
- Reduces RAM access frequency
- Essential for high-performance computing

**Example:**
If CPU needs data from address 1000:
- Without cache: 100 nanoseconds (fetch from RAM)
- With cache (hit): 1 nanosecond
- **100x faster!**

### 4. VLOOKUP Function (1 mark)

**Purpose:** Search for value in first column of table, return value from another column
